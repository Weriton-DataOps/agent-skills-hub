# Atelier — o ateliê de design do OverCore

O **Atelier** é o mestre de ateliê: dirige a obra e convoca **artesãos hiper-estreitos**
(um só faz estados de feedback, outro só microcopy, outro só landing pages). O roster
não é fixo — ele **DESCOBRE** (skill existente no hub) → **IMPROVISA** (artesão efêmero
quando não existe) → **CRISTALIZA** (o que valeu vira skill permanente via curadoria).

- **[`CLAUDE.md`](CLAUDE.md)** — o corpo do mestre: identidade, herança inegociável,
  roster vivo, contrato do artesão, wizard com gates, registro (landing/app/docs),
  **18 regras anti-slop determinísticas** e o vocabulário (`/croqui`, `/refinar`,
  `/artesao`, `/varrer`, `/cristalizar`…).
- **[`style-gallery.html`](style-gallery.html)** — a galeria de direções visuais
  (glassmorphism, skeuomorphism, neo-brutalism, claymorphism, minimalism, liquid glass),
  cada uma com exemplo ao vivo. Etapa 1 do wizard quando não há design system.
- **[`AUDITORIA.md`](AUDITORIA.md)** — a auditoria do arsenal (69 skills lidas pelo
  conteúdo): roster titular por etapa, 27 artesãs existentes, 14 buracos, cortes aplicados.

## O fluxo em uma linha

```
herança → direção (galeria) → DESIGN.md → croqui → [gate] → peça-a-peça (artesãos) → varredura anti-slop → [gate] → código
```

Nenhum código nasce antes de refinado aprovado por humano. Nenhum token existente é
sobrescrito. Nenhum artesão entrega peça que viole as regras A1–A18.
